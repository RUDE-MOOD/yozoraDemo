<h2>プロジェクト概要</h2>
<p>本アプリはデジタル社会における過度な承認欲求や他者評価への依存による精神的疲弊を軽減し、ユーザーが自分自身と静かに向き合える「個人的な記録空間」を提供することを目的とする。
「いいね」や数値評価を排除した設計により、他人の視線から解放された安心できる場所を実現する。
<br>
<br>
また、日常の成功・失敗や「何もしなかった一日」も含め、すべての経験を価値ある人生の一部として可視化することで、ユーザーが自身の歩みを肯定的に捉え直し、自己肯定感を回復・維持するきっかけを創出する。
<br>
<br>
ユーザーの感情や思い出を「星」として可視化し、それらが漂うデジタルプラネタリウム空間で蓄積・再訪できるアプリケーションである。日記のようにユーザーが文章を記入しスライダーでその日の気持ちを入力し星として打ち上げられ、星空に表示する。テキストとスライダー入力を通じて得られた情報を解析し、感情の状態や意味的な関連に応じて、星の位置・色・輝度・動きとして表現する。時間とともに緩やかに移動・集まり、最終的に星座を形成していく。
<br>  
<br>
情報の記録・整理・振り返りを「管理」ではなく「眺める体験」へと変換するというポイント。
</p>

<h3>なぜプラネタリウムなのか</h3>
<p>星空は独立した点の集まりでありながら、距離や配置によって意味が生まれ、見る人によって解釈が変わるとういう性質を持つ。人の感情や記憶の在り方と非常に似ている。星は評価されない存在であるため、正解がない、比較されない、現代の多くのオンラインサービスと違い、承認要求を満たすという目的ではなく、ユーザーの感情や思い出をコンテンツとして扱わず、評価や最適化の対象にしないという思想で、星空を使ったメタファーが自然に支えている。</p>

<h3>想定ユースケース</h3>
<p>仕事帰りに疲れていてSNSを見る気にならない。今日は面白い出来事があったが他人に見せるほどではないけど残しておきたい気持ちはある。寝る前にノスタージアでそれを自分のそらに打ち上げていつか自分の励ましになると思って心が落ち着く。その日の振り返りをしながら星座になりかけている星を眺めて当時の思い出を思い浮かぶ。<p>

  <ul>

  </ul>
  <p>ほとんどのユーザーがモバイル端末で本サービスを利用することが想定されているため、処理の軽量化・タッチ操作メインとういうデザイン方針。なるべく処理はサーバーで行いたいのでNode.jsを使用して計算とデータの管理をする。更にビジュアルは全部3Dでレンダーするではなく、ユーザーが操作できる要素のみを3D描画する。他はThree.jsでシェーダーを使うことで少ないGPU負荷で綺麗なビジュアルが実現できる。Hugging Faceのベクター化モデルがオープンソースであるため、カスタム性が高く無料で使用できる。</p>

<h2>レイヤー構造</h2>
<h4>プラネタリウム自体が７つのレイヤーでできている。</h4>
<ol>
  <li>背景（宇宙の色）<br>Three.jsのシェーダー。自由に変えられるようにするため画像を使用しない。</li>
  <li>流体層（天の川風）<br>GPUへの負荷の最小限に抑えるため、ノイズシェーダーを使用。UVマップをゆっくりずらすことで動いているように見せる。</li>
  <li>背景の遠い星 <br>ユーザーが操作しないのでノイズマップですませる。</li>
  <li>霧 <br>さらに立体感を出すためのレイヤー。3Dボリュームでやるとすごく重たくなるのでこれもノイズマップで実現する。深度に応じて透明度の変化もできる。</li>
  <li>星（ユーザーが操作できる）<br>3Dで描画する。ユーザーの感情やその日の重要度に応じて輝度、大きさ、色が変化。レンダーしないグリッド上に存在する。ユーザーが押下するとその思い出のメニューが出てくる。</li>
  <li>ビネット <br>画面の角を暗くすることで立体感が出る。</li>
  <li>UI描画 (ユーザーが操作できる）<br>メニューやボタンのような要素を描画するレイヤー</li>
</ol>
<p>こういう風にレイヤー毎に分けると、各要素を独立させレイヤー同士の直接依存が生じず拡張性を高めることができる。レイヤー毎の描画トグル・テーマの切り替え・低スペック端末向けのパフォーマンス描画モードも実装可能。</p>
